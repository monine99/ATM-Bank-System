# ATM 银行系统 v2.0 — 使用指南

## 项目简介

基于 **TCP Socket** 通信的 ATM（自动取款机）客户端-服务器程序。服务器端模拟银行后台，处理多个 ATM 客户端的并发请求；客户端模拟 ATM 终端，提供数字菜单交互界面。

通信协议遵循 **RFC-20242024**。

### v2.0 新特性

- **存款 (DEPO)** + **转账 (TRAN)** 功能
- 客户端数字菜单选择，无需手输命令
- 密码安全输入（Console 环境下自动隐藏回显）
- 服务器端实时监控面板（30秒间隔输出）
- 完整交易日志记录 (`transactions.log`)
- 金额使用 `BigDecimal` 精确计算

## 快速开始

### 第一步：编译程序

打开终端，进入项目目录，执行：

```bash
cd ATM/
javac ATMServer.java ATMClient.java
```

### 第二步：启动服务器

```bash
java ATMServer
```

默认监听端口 **2525**，如需指定端口：

```bash
java ATMServer 8888
```

> 端口范围必须为 1024~65535。

启动成功显示：

```
==============================================
  ATM 银行服务器已启动
  监听端口: 2525
  按 Ctrl+C 停止服务器
==============================================
```

每 30 秒服务器会自动输出运行状态面板。

### 第三步：启动客户端

打开**新终端**，进入同一目录：

```bash
java ATMClient
```

默认连接 `127.0.0.1:2525`。连接远程服务器：

```bash
java ATMClient 192.168.1.100 2525
```

## 通信协议说明 (RFC-20242024)

### 客户端 → 服务器

| 命令 | 格式 | 说明 |
|---|---|---|
| HELO | `HELO <卡号>` | 提交卡号，发起认证 |
| PASS | `PASS <密码>` | 提交密码，完成认证 |
| BALA | `BALA` | 查询当前余额 |
| WDRA | `WDRA <金额>` | 取款，金额需 > 0 |
| DEPO | `DEPO <金额>` | **新增** — 存款，金额需 > 0 |
| TRAN | `TRAN <卡号> <金额>` | **新增** — 转账给其他账户 |
| QUIT | `QUIT` | 退出登录，断开连接 |

### 服务器 → 客户端

| 响应 | 含义 |
|---|---|
| `500 AUTH REQUIRE` | 卡号有效，要求输入密码 |
| `525 OK!` | 操作成功 |
| `401 ERROR!` | 操作失败（余额不足/密码错误/未认证/卡号不存在） |
| `AMNT:XXXX.XX` | 查询余额的返回结果 |
| `BYE` | 会话结束，客户端应断开连接 |

### 状态机

```
[初始] --HELO(卡号存在)--> [等待密码] --PASS(正确)--> [已登录]
  |                             |                        |
  +--- HELO(卡号不存在)         +--- PASS(错误)           +--- BALA → 返回余额
       → 401 ERROR!                 → 401 ERROR!          +--- WDRA → 取款
                                                          +--- DEPO → 存款
                                                          +--- TRAN → 转账
                                                          +--- QUIT → BYE
```

## 组内测试

两台主机需在同一局域网内。

| 角色 | 操作 |
|---|---|
| 主机 A (服务器) | `java ATMServer` |
| 主机 B (客户端) | `java ATMClient <A的IP地址> 2525` |

主机 A 查看 IP: Windows 执行 `ipconfig`，找到 IPv4 地址。

如果主机 B 连不上，在主机 A 以管理员身份放行防火墙：

```bash
netsh advfirewall firewall add rule name="ATM Server" dir=in action=allow protocol=TCP localport=2525
```

## 技术要点

- **并发安全**: `ConcurrentHashMap` + `synchronized` 方法保证多线程安全
- **金额精度**: 使用 `BigDecimal` 替代 `double`，避免浮点误差
- **I/O 模型**: `BufferedReader` / `PrintWriter` 行协议通信
- **资源管理**: try-with-resources 自动关闭 Socket 和 I/O 流
- **状态机**: 命令解析与业务逻辑分离，认证状态驱动
- **监控线程**: daemon 线程定期输出服务器运行状态
